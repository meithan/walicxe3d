25 !> @brief Benchmarking functions
26 !> @details Contains functions and subroutines designed to measure
27 !! the execution time between points in the program execution.
28 !> @section changelog Changelog
29 !! @li @b 1.5
30 !! @c TOC now returns a real with the number of elapsed seconds, and two
31 !! new functions were added: @c FTIME accepts a real representing a
32 !! time span in seconds, and returns a character string with hours, minutes
33 !! and seconds, as needed. @c NICETOC receives an @c INTEGER time mark, passes it
34 !! to toc, and then returns the result of timeformat, i.e., a formatted
35 !! character string represented the time elapsed.
36 !! @li @b 1.4
37 !! Some subroutines were converted into functions that return
38 !! a character string instead of printing the result to screen.
39 !! @li @b 1.3
40 !! The subroutine @c ETA was added. It allows to estimate the remaining
41 !! execution time based on a user provided progress estimator.
42 !! @li @b 1.2
43 !! The intrinsic @c SYSTEM_CLOCK is now used instead of
44 !! @c DATE_AND_TIME to calculate time intervals. See Comments.
45 !! @li @b 1.1
46 !! @c STAMP was added to generate nicely formatted timestamps.
47 !! @li @b 1.0
48 !! First version.
49 !> @section comments Comments
50 !! Starting from version 1.2, the module utilizes @c SYSTEM_CLOCK
51 !! instead of @c DATE_AND_TIME because it provides higher time
52 !! resolution. The disadvantage of this is that @c SYSTEM_CLOCK
53 !! has a relatively small looparound period, which is compiler-dependent.
54 !! For the GCC 4.2.1, this value is 24.855 days, so trying to measure
55 !! a time interval larger than this will produce incorrect results.
56 !! @n @n This program measures wall time.
57 !! @section usage Module Usage
58 !! Time measurements are done with the function pair @c TIC and @c TOC.
59 !! One must first call @c TIC to mark a certain point in the execution
60 !! of the program, and then @c TOC can be called at any later time in
61 !! order to calculate the elapsed time since the moment marked by @c TIC.
62 !! @n @n
63 !! Subroutine @c TIC modifies the given @c INTEGER argument @c mark storing
64 !! information about the point in time when it was called. Multiple
65 !! time marks (one variable per mark) can be used in the program to mark
66 !! different points. Time resolution is system dependent but is generally
67 !! on the order of milliseconds.
68 !! @n @n
69 !! The @c ETA function estimates the execution time remaining. It must
70 !! be given a real number between 0 and 1 that indicates the percentage
71 !! progress of the execution at the moment of the call, as well as a time
72 !! mark obtained at the program start.
73 !! @n @n
74 !! The @c STAMP function returns a string with the current date and time.
75 !! It is common to call @c STAMP together with a call to @c TOC.

172  !> @brief Computes the estimated remaining time
173  !> @details Returns the estimated remaining time of execution.
174  !! This requires a time mark previously returned by @c TIC marking
175  !! the start of execution (or, alternatively, a zero value), as well
176  !! as a progress value given as a real number between 0 and 1.
177  !> @return A character string with the hours, minutes and seconds remaining.
178  character(len=20) function eta (mark, progress)
179  integer, intent(in) :: mark
180  real, intent(in) :: progress
181 
182  integer :: now, elapsed, rate, cmax
183  integer :: hours, mins
184  real :: remaining, secs
185  character(len=20) :: buffer
186 
187  call system_clock(now,rate,cmax)
188 
189  elapsed = now - mark
190  remaining = elapsed/progress - elapsed
191 
192  hours = int(remaining/real(rate)/3600.)
193  mins = int(remaining/real(rate)/60.) - hours*60
194  secs = remaining/real(RATE) - hours*3600 - mins*60
195 
196  if ((hours==0).and.(mins==0)) then
197  201 format(f6.3,'s')
198  write(buffer,201) secs
199  elseif (hours==0) then
200  202 format(i2,'m ',f6.3,'s')
201  write(buffer,202) mins, secs
202  else
203  203 format(i4,'h ',i2,'m ',f6.3,'s')
204  write(buffer,203) hours, mins, secs
205  end if
206 
207  eta = buffer
208  return
209 
210  end function
